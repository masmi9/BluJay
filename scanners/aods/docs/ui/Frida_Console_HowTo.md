## Frida Console How‑To

### Prerequisites
- UI reachable (dev): `http://127.0.0.1:5088`
- API reachable: `http://127.0.0.1:8088`
- Auth token present (app will seed from login; tests seed via `aodsAuth`)

### Header Overview
- Selected device chip:
  - Color reflects status: attached (green), port closed (red), available (blue), unavailable (amber)
  - Forwarded‑port badge (dot) shows TCP 27042 open/closed on forwarded devices
- Execution/Determinism/Calibration chips summarize runtime policy and model calibration
- “more” menu: Ensure (ADB+Frida), Refresh health/status, Reset environment, Diagnose connectivity

### Reset Environment
Use when state looks inconsistent (e.g., device list stale, events frozen).

Steps:
1) Click “more” → “Reset environment”
2) The console closes WS/SSE, clears page state/local storage (keeps auth), and reloads health/devices

Acceptance:
- WS status becomes “disconnected”; SSE indicator off
- Device/Process tables clear and repopulate
- Header chips rehydrate (Mode/Port/PID)

### Diagnose Connectivity
Runs a quick checklist with actionable messages.

Checks:
- API `/health`
- Frida health (execution mode, port 27042, frida‑server PID, server version)
- Devices list
- WS token mint (policy‑aware; 409 in static‑only)

Steps:
1) Click “more” → “Diagnose connectivity”
2) Review results; use “Copy diag” where provided

Acceptance:
- Each row shows OK/FAIL with a brief note
- Static‑only mode shows WS token 409 without throwing

### Preflight Guards (Safety/Policy)
Protect dynamic actions with clear, inline feedback.

Guards:
- Static‑only policy: blocks Attach/Upload/WS, shows info banner and warning toast
- Auth required: blocks API‑backed actions until authenticated
- Selected device required: blocks attach/upload without a target
- Port 27042 required for upload: blocks upload if closed, suggests Ensure

Acceptance:
- In static‑only, “Attach/Upload/Connect WS” are disabled or emit policy warning
- Upload shows: “Port 27042 closed. Try Ensure (ADB+Frida).” when port is closed

### Deep‑links (Shareable State)
Supported query params:
- `device` (e.g., `local`, `socket@127.0.0.1:27042`)
- `pkg` (package name, e.g., `com.acme.app`)
- `rpcTab` (0=Result, 1=Status, 2=Raw)
- `compact` (`1` for compact header)
- `eventsFilter` (substring to filter Live Events)

Examples:
```
http://127.0.0.1:5088/frida?device=local&pkg=com.acme&rpcTab=2&compact=1&eventsFilter=ssl
```

Acceptance:
- On load, header reflects device/mode; RPC tabs/selectors reflect params
- URL updates when changing device/pkg/tab/compact/filter

### Live Events (Performance)
- Virtualized list (react‑window) caps DOM nodes (< 300) even for large streams
- Auto‑scroll to tail and pause‑on‑error toggle

Acceptance:
- With >1000 events, DOM nodes in `#events-panel` remain < 300
- “Pause on error” stops auto‑scroll after an error line arrives

### Favorites & RPC Presets
- Favorite devices/processes pin to the top and persist
- RPC presets support search and tag filter; “Save as preset” stores current call

Acceptance:
- Toggling star on device/process persists and reorders
- Preset search/tag filter narrows options; running preset populates fn/args and executes

### Accessibility
- Chips/buttons labeled: `execution-mode`, `static-only`, `sse-status`, etc.
- Live region for events with `role="log"`, `aria-live="polite"`

Acceptance:
- Screen readers announce WS/SSE changes; tab order starts at Devices controls
- Contrast ≥ 4.5:1 for chips in light/dark themes

### Troubleshooting Tips
- Port badge red (27042 closed): run “Ensure (ADB+Frida)” and try “Refresh health/status”
- Static‑only: dynamic tools disabled by policy; switch mode to enable
- “Unavailable but frida‑server started”: refresh health; forwarded‑port badge helps disambiguate

### Quick Reference
- Refresh health/status: “more” → “Refresh health/status” (or header refresh)
- Reset environment: “more” → “Reset environment”
- Diagnose connectivity: “more” → “Diagnose connectivity”
- Copy device diag: open Selected chip popover → “Copy diag”
